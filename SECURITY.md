# Security Policy

## Supported Versions

Security fixes are planned for the latest tagged minor release once stable releases begin.

Until a `1.0.0` release is tagged, please treat the `main` branch as the active development line.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub private vulnerability reporting when available:

https://github.com/FrozonFreak/laravel-plan-manager/security/advisories/new

If private reporting is unavailable, contact the maintainer through the GitHub profile and provide:

- A concise description of the issue.
- Reproduction steps or proof of concept.
- Impact and affected versions or commits.
- Any suggested remediation.

This package does not process payments and should not store payment card data. Billing providers remain responsible for payment security.
